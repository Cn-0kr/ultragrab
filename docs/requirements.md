# 产品需求分析 - 万能视频下载站

## 一、产品定位

一个覆盖 1800+ 视频平台的**在线视频/音频/字幕下载工具**，基于开源项目 `yt-dlp` 封装。第一版目标是**可演示、可本地运行、可直接走商业化**的 MVP。

- 不做自研解析器，依赖 yt-dlp 的能力。
- 以单页 Web 应用形式对外，桌面 + 移动双端可用。
- 明确付费价值点：批量、高清、字幕、合并、省时间、手机可用。

## 二、目标用户

| 用户类型 | 典型场景 | 付费意愿点 |
| --- | --- | --- |
| 自媒体 / 内容二次创作 | 批量采集素材（YouTube/TikTok/X/Bilibili）、需要最高画质+字幕 | 批量、高清、字幕下载 |
| 学习/资料收藏党 | 保存课程、播客、演讲视频到本地 | 合并音视频、字幕、离线观看 |
| 出海/跨境团队 | 需要下载海外视频做本地化 | 代理稳定、字幕翻译（Pro） |
| 普通移动端用户 | 手机看到一个视频想保存 | 手机端可用、粘贴即下载 |

非目标用户：希望绕过 DRM、付费墙、会员限制的用户；批量爬虫 / 重度商业抓取用户。

## 三、核心使用场景（User Stories）

1. 作为用户，我粘贴一条视频 URL，1 秒内看到视频标题、封面、时长、可选清晰度。
2. 作为用户，我可以选择清晰度（720p / 1080p / 4K）和下载模式（服务端合并 / 直链代理 / 直链重定向）后一键下载。
3. 作为用户，我可以一次粘贴多条链接，批量解析并展示每条的状态。
4. 作为用户，我可以查看该视频的字幕语言列表，选择下载原字幕或自动字幕。
5. 作为用户，我在手机浏览器上也能完成以上操作，下载按钮易点，状态清晰。
6. 作为用户，遇到失败时能看到明确原因提示（地域受限 / 需登录 / 平台不支持 / 直链过期）。

## 四、功能清单

### 4.1 Must Have（MVP 必须）

- [M1] 单链接解析：返回标题、封面、时长、格式列表（清晰度/大小估计/是否含音频）、可用字幕语言。
- [M2] 批量链接解析：同时支持多条 URL，逐条返回结果，单条失败不影响整体。
- [M3] 三种下载模式：
  - `server`：后端 yt-dlp 下载到临时目录，必要时 ffmpeg 合并音视频/嵌字幕，返回内部下载链接。
  - `proxy`：后端代理直链，透传 Range 与 yt-dlp 给出的 `http_headers`（Referer/UA）。
  - `redirect`：后端 302 到直链（轻负载，Referer 强校验的平台可能失败，前端提示并建议回退）。
- [M4] 任务状态查询：`queued / downloading / done / error`、进度百分比、下载地址、错误原因。
- [M5] URL 合规校验：仅允许 http/https，拒绝本地/内网/file 协议。
- [M6] 明确合规边界：前端 footer + FAQ 声明仅用于用户有权下载的公开/授权内容，不绕过 DRM/付费墙。
- [M7] 前端首页：Video School 风格骨架（首屏 URL 输入 + 批量 + 解析按钮 + 会员卖点 + FAQ）。
- [M8] 前后端联调走通端到端（阶段 3 验收）。

### 4.2 Should Have（第一版建议）

- [S1] 字幕下载：选择语言后触发服务端合并或单独下载字幕文件。
- [S2] 下载进度：服务端模式挂 `progress_hooks`，前端轮询显示百分比 + 速度 + ETA。
- [S3] 浏览器历史：`localStorage` 缓存最近任务，支持重试、清空。
- [S4] 失败友好提示：将 yt-dlp 原始错误映射成用户语言（地域受限、登录、平台不支持、直链过期）。
- [S5] 临时文件自动清理：超过保留时间或任务完成一段时间后删除。

### 4.3 Could Have（后续 Pro 付费能力）

- [C1] 字幕翻译（接翻译 API）。
- [C2] AI 摘要（视频转录 + 总结）。
- [C3] 用户系统 + 会员 + 支付（Stripe / Creem / 国内支付）。
- [C4] 云端历史 / 跨设备同步。
- [C5] 去水印、格式转换（mp3 / mp4 / webm）。
- [C6] 浏览器扩展 / 桌面客户端。

### 4.4 Won't Have（明确不做）

- [W1] 不绕过 DRM、付费墙、平台访问控制。
- [W2] 不做账号代登录（不替用户登录 YouTube/Bilibili）。
- [W3] 不做大规模爬虫/聚合资源站，不存储解析后的视频二次分发。

## 五、非功能性需求

| 维度 | 目标 |
| --- | --- |
| 性能 | 单条解析 P95 < 5s；服务端下载 1080p 10 分钟视频 P95 < 90s（受网络影响） |
| 可用性 | MVP 单机可用；支持同时在线下载 ≥ 3 个并发任务 |
| 安全 | URL 协议白名单；内网地址拒绝；临时文件 ID 化命名；代理仅接受服务端已签发的 URL |
| 可观测 | 后端结构化日志；任务状态可查询；错误可追溯到具体 URL + 阶段 |
| 可维护 | yt-dlp 不改源码，只做封装；前端与后端独立启动；文档与实现保持同步 |
| 部署 | 可本地 `uvicorn` + `vite dev` 跑通；生产可 Nginx 反代 + Systemd / Docker 后续补 |

## 六、合规边界

- 产品定位为"用户有权下载的视频的下载工具"，类似浏览器右键保存。
- 首页 / FAQ / Footer **必须**出现合规声明：仅用于合法授权或公开内容，用户应自行确认版权。
- 不提供绕过 DRM、付费墙、会员限制的能力。
- 不做内容聚合、二次分发、不缓存用户下载的原始内容超过必要时间。
- yt-dlp 本身是开源项目，我方仅作为其调用方，遵循其许可证。

## 七、成功指标（MVP）

- 能在本地 `uvicorn + vite` 环境下跑通 **3 种下载模式** × **≥ 2 个主流平台**（YouTube、Bilibili、X/Twitter 任选）。
- 桌面（1440px）和移动（390px）两个断点下首页可读、按钮可点、无横向滚动。
- 常见失败场景（非法 URL / 不可下载 / 直链过期）有明确提示，不卡住界面。
- 文档（requirements / design / plan）与代码结构一致，可供新 AI 接盘继续迭代。

## 八、范围外但要预留的扩展点

- Pro 会员：价格展示 + 占位 CTA，不接真实支付。
- 字幕翻译 / AI 摘要：前端 UI 位预留 "Pro only"，后端暂不实现。
- 登录态 / Cookie 透传：yt-dlp 支持 `cookies-from-browser`，后续管理面再暴露。
- 队列持久化：当前用内存，后续可替换 Redis / SQLite。

## 九、维护节奏

- 范围调整必须改本文件，并同步 `design.md` / `plan.md`。
- 每阶段验收时复核 Must/Should 进度，写入 `changelog.md`。
